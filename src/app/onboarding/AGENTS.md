# Onboarding Page Instructions

## Scope

This folder owns the curriculum-aware onboarding experience.

## Product Rules

- Onboarding must collect Subject, Language, Level, and Exam Session in a way that supports IB Diploma Programme workspace generation.
- Preserve the requirement for six selected subjects unless the product requirements explicitly change.
- Do not replace curriculum-aware setup with a generic folder or notes tree.
- Completed onboarding must create subject workspaces, IA structures, syllabus scaffolding, and core programme structures through the existing API/helper flow.

## UI Rules

- Read the Synapse design skill before visual changes.
- Keep the first-run flow focused on setup work, not marketing copy.
- Make validation errors specific enough that the student can correct their selections.
